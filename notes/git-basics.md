# 📚 Шпаргалка по Git: Базовые команды

## ⚙️ 1. Первоначальная настройка
Эти команды нужно выполнить один раз, чтобы ваши коммиты были правильно подписаны.

* `git config --global user.name "Ваше Имя"` — задает имя автора.
* `git config --global user.email "ваш@email.com"` — задает email автора.

---

## 🚀 2. Создание и получение репозитория
С этого начинается работа над любым проектом.

* `git init` — создает новый локальный репозиторий в текущей папке.
* `git clone <url-репозитория>` — скачивает существующий проект (например, с GitHub или Bitbucket) на ваш компьютер.

---

## 🔄 3. Повседневный рабочий процесс
Самые частые команды, которые используются постоянно при написании кода.

* `git status` — **ваш лучший друг**. Показывает текущее состояние: какие файлы изменены, какие добавлены в индекс (готовы к сохранению), а какие нет.
* `git add <имя-файла>` — добавляет конкретный файл в "индекс".
* `git add .` — добавляет **все** измененные и новые файлы в текущей директории.
* `git commit -m "Добавил экран регистрации"` — сохраняет подготовленные файлы в историю с понятным описанием того, что было сделано.

---

## 🌿 4. Работа с ветками (Branches)
Ветки позволяют безопасно разрабатывать новые фичи, не ломая основную рабочую версию проекта.

* `git branch` — показывает список всех локальных веток. Текущая отмечена звездочкой `*`.
* `git branch -vv` — **расширенный статус веток**. Показывает локальные ветки, их привязку к удаленным (remote-tracking), последний коммит и статус (отстает ли ветка, ушла ли вперед, или удалена на сервере — `[gone]`).
* `git branch <имя-ветки>` — создает новую ветку, но остаётся на текущей.
* `git checkout <имя-ветки>` (или `git switch <имя-ветки>`) — переключает вас на указанную ветку.
* `git checkout -b <имя-ветки>` (или `git switch -c <имя-ветки>`) — создает новую ветку и **сразу** переключается на неё (самый частый и удобный вариант).
* `git merge <имя-ветки>` — вливает изменения из указанной ветки в ту, на которой вы сейчас находитесь.

---

## ☁️ 5. Удаленные репозитории (Remote)
Команды для синхронизации локального кода с удаленным сервером.

* `git push origin <имя-ветки>` — отправляет ваши локальные коммиты на сервер.
* `git pull` — скачивает новые изменения с сервера и сразу пытается влить их в вашу текущую рабочую ветку.
* `git fetch` — скачивает информацию об изменениях на сервере, но **не меняет** ваши локальные файлы (безопасный просмотр того, что произошло в облаке).

---

## ⏪ 6. Просмотр истории и отмена изменений
* `git log` — показывает историю коммитов. Для компактного вывода используйте `git log --oneline`.
* `git restore <имя-файла>` — отменяет локальные изменения в файле, возвращая его к состоянию последнего коммита (работает, если вы еще не сделали `git add`).

## 🗑️ 7. Удаление веток и очистка (Clean up)
Команды для поддержания порядка в репозитории и удаления старого кода.

* `git branch -d <имя-ветки>` — безопасное удаление локальной ветки. (Git выдаст ошибку, если в ветке есть код, который вы еще никуда не сохранили/не слили).
* `git branch -D <имя-ветки>` — **принудительное** удаление локальной ветки. Уничтожает ветку вместе со всеми изменениями в ней.
* `git fetch --prune` (или `git fetch -p`) — скачивает новые данные с сервера и стирает "закладки" на ветки, которых на сервере больше нет.
* `git branch -vv | grep ': gone]' | awk '{print $1}' | xargs git branch -d` — магическая команда: находит все ваши локальные ветки, которые были удалены на сервере (имеют статус `[gone]`), и массово их удаляет.


## 🌳 8. Просмотр истории (Логи)
Как понять, кто, что и когда делал в репозитории.

* `git log` — стандартный вывод всей истории коммитов (чтобы выйти из режима просмотра, нажмите клавишу `q`).
* `git log --oneline` — компактный вид: один коммит = одна строчка (только короткий хэш и сообщение).
* `git log --graph --oneline --all --decorate` — **красивое дерево веток**. Рисует схему слияний и ответвлений прямо в терминале, показывает, где находится `main`, удаленные ветки (`origin/main`) и ваши локальные фичи.
* `git log -p <имя-файла>` — показывает историю изменений конкретного файла (сразу видно, кто и какие именно строчки добавлял или удалял).
* `git show <хэш-коммита>` — показывает детальную информацию о конкретном коммите (какие файлы изменились и как именно).

### 💡 Супер-лайфхак: Создаем алиас (сокращение)
Команду для отрисовки красивого дерева (git log --graph --oneline --all --decorate) нереально набирать каждый раз вручную. В Git можно создавать свои собственные короткие команды!

Выполните в терминале вот эту команду (один раз):
```bash
git config --global alias.tree "log --graph --oneline --all --decorate"
```

Что это даст:
Теперь Git запомнил это сокращение. В любой момент, в любом вашем проекте, чтобы увидеть красивое дерево веток, вам достаточно будет написать просто:
```bash
git tree
```
